## State / handler / latency architecture

```
                       ┌──────────────────────── app/page.tsx (client island) ───────────────────────┐
                       │                                                                              │
  user types ──▶ ApplicantForm ──onChange──▶ react-hook-form store ──watch()──▶ applicant values     │
                       ▲                                   │                                          │
                       │ inline zod errors (422 mirror)    ▼                                          │
                       │                        useDeferredValue(values)   (React 19 — keep typing snappy)
                       │                                   │                                          │
                       │                                   ▼                                          │
                       │                    useEvaluate(deferredValues)                               │
                       │                    ├─ useDebouncedValue(250ms)                               │
                       │                    ├─ React Query: queryKey=['evaluate', values]             │
                       │                    │    • keepPreviousData: true   (no flicker)              │
                       │                    │    • AbortController via signal (cancel stale)          │
                       │                    │    • staleTime small; no retry on 422                   │
                       │                    └─ returns { data, isFetching, error }                    │
                       │                                   │                                          │
                       │                                   ▼                                          │
                       └────────────────────── RuleMatchPanel (React.memo) ◀── selected bank state ──┘
                                                │
                          ┌─────────────────────┼───────────────────────────┐
                          ▼                     ▼                            ▼
                   StatusBadge          EligibleBanks (chips)        BankRuleList
              (eligibility_status)    (data.eligible_banks)     (ConfidenceBar + RuleRow[]
                                                                  for evaluations[selected])
```

**Ownership (one owner per kind of state):**
- **Form state** → react-hook-form (the 23 inputs + validation).
- **Server state** → React Query (`useEvaluate`): the `EvaluateResponse`, fetching/error flags, caching, cancellation.
- **UI state** → local `useState` (selected bank, expanded groups). No global store needed for v1; add **zustand** only if a sibling needs the result outside the page tree.

**Handler tree (events → effects):**
- `onFieldChange` → RHF updates value → `watch()` emits → `useDeferredValue` → debounced query key changes → React Query (re)fetches (cancels the previous) → `data` updates → memoized panel re-renders.
- `onSelectBank(bankName)` → local `setSelectedBank` → `BankRuleList` re-renders only.
- `onError(422)` → map `error.details.errors[*].loc` → set RHF field errors inline.

**Latency tactics (the whole point — "quick response"):**
1. **Debounce 250 ms** so we don't fire on every keystroke.
2. **`keepPreviousData`** so the panel shows the last good result while the next loads (zero flicker).
3. **`AbortController`** (React Query per-key) cancels superseded requests — only the latest wins.
4. **`useDeferredValue` + `React.memo`** keep typing at 60fps while the result tree re-renders off the critical path.
5. **Stable selectors / narrow props** so editing `income` doesn't re-render unrelated rule rows.
6. **Next rewrite proxy** (`/api/backend/*`) → same-origin, no CORS preflight, backend URL stays server-side.
7. First load → **skeleton**; thereafter seamless. Backend is ~µs, so perceived latency = network only.

## Critical risks / decisions
- **Next 16 API drift** → read the bundled docs first (per `AGENTS.md`).
- **CORS** → solved by the rewrite proxy; if calling the backend directly instead, add CORS middleware server-side.
- **Tailwind v4** → tokens are CSS `@theme`, *not* a JS config; utilities derive from the vars.
- **Don't over-store** → RHF + React Query cover v1; resist a global store until proven necessary.
